I remember the first time I stumbled across Placehold.it, a simple image placeholder serving gray rectangles with text indicating the width and height. Immediately, I wondered how the site was able to deliver an infinite number of custom images. Do they generate all their images on the fly? Do they cache the images on the server?

I can't attest to Placehold.it's serverside logic, but I can give a technical overview of the image placeholder I built to learn how an image placeholder can work.

How it works

Chartholdr is built with the Ruby on Rails framework. Taking advantage of the MVC routing, I grab the graph type, width, height (optional), and color (optional) all from one URL comprised of the following syntax. http://chartholdr.io/bar/1000/500/c=9d5ddd The routes.rb calls the Application Controller which handles creating the generator and calling the send_data method which sends the inline png file. The generator's constructor takes in the chart object so it knows which templates to use when rendering. If the image isn't already cached in the public folder, the generator's get method initializes PhantomJS, renders the JS chart, saves the image to the public folder and then returns the file location. ruby bar = Chart.new("bar", @width, @height, @color) generator = Generator.new(bar, @force_cache) send_data(File.open(generator.get).read, :type => "image/png", :disposition => 'inline')
